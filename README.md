# 编译技术实验
## 实验一：词法分析器
### 实验目的
1、	学会针对DFA转换图实现相应的高级语言源程序。
2、	深刻领会状态转换图的含义，逐步理解有限自动机。
3、	掌握手工生成词法分析器的方法，了解词法分析器的内部工作原理。


### 项目结构
lexer.h: 定义所需的token类型(保留字,特殊字符,其他等),定义词法分析器状态及所需函数声明

lexer.c: 词法分析器的具体实现:
        1.识别c语言的单词;
        2.自动去除空白,制表符,换行符;
        3.自动识别并忽略注释;
        4.识别完成输出对应的属性二元组;
        5.对有问题的识别自动报错;

main.c: 词法分析器的运行主函数:
        1.输出源程序;
        2.输出对应的二元组;
        3.输出报错信息;

test1.c: 测试文件
### 运行方式
在终端内编译相关文件:**gcc lexer.c main.c -o lexer**
使用命令运行: **./lexer test1.c **   

**测试结果存放在result1.txt中**

## 实验二：递归下降语法分析器
### 实验目的
首先**对上下文无关文法进行检查**，**消除左递归和左公共因子**，从逻辑上检测避免死循环和低效率处理。 采用每个产生式的**左边的文法符号**对应一个函数或过程的形式，编写程序实现一个递归下降分析器。

### 项目结构
使用了实验一的词法分析器;

parser.c: 递归下降语法分析器的实现

parser_main.c:语法分析器的运行主函数

test2.c: 测试文件

### 运行方式
编译：**gcc lexer.c parser.c parser_main.c -o parser**
运行：**./parser test2.c**

**测试结果存放在result2.txt中**
